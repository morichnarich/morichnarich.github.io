Dockerイメージとコンテナの削除方法

2021/01/06 2021/01/06
カテゴリー programming

結論からいうと、コンテナを削除してからイメージを削除するのがきれいです。

    ターミナル

    コンテナの削除

    コマンド： docker rm [コンテナID]

    Docker

    複数指定もできる

    コマンド： docker rm [コンテナID-a] [コンテナID-b]

    Docker

    それでも消えない場合

    force(-f)を付与すれば削除することができる。

    コマンド： docker rm -f [コンテナID]

    Docker

    消えていることを確認

    ターミナル

    裏技

    docker ps -a -q は、コンテナIDの一覧をだしてくれるので、それと合わせると一括削除もできる。

    コマンド： docker rm `docker ps -a -q`

    Docker

    イメージの削除方法

    現状のコンテナの確認

    ターミナル

    現状のイメージの確認

    ターミナル

    イメージの削除

    コマンド： docker rmi [イメージID]

    Docker

    コンテナが存在しているというメッセージが表示された場合

    forceオプションをつけてみると消える。（ただし、コンテナは残るみたい。）

    コマンド： docker rmi -f [イメージID]
