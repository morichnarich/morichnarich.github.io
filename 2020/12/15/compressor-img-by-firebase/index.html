Cloud Functions for FirebaseとGmailを使って、各デバイスで作成した圧縮画像を別のデバイスでも気軽に利用できるwebサービスを作る

2020/12/15 2020/12/28
カテゴリー programming

ブログのネタ作りにおいて画像は欠かせない表現手段ですが、画像が重たいとブログ自体も重たくなってきます。時短大好きな私としては、撮ったりネットで見つけたりした画像が重たい場合にGmailを利用して、例えば、iPhoneで撮影した画像をfirebaseアプリで画像を圧縮したファイルオブジェクトを作ってcloud firestoreに保存、gmailにメールで通知し、gmailのリンクをから、ダウンロード用のブラウザを立ち上げ、ダウンロードすることで圧縮画像を取り込む形を作ります。gmailを使うことでどのデバイスでも圧縮画像を保存できるため、自宅で画像を圧縮する手間が省けます。

それでは作ってみましょう！

ポイントはfunctions.httpsCallable。Cloud Functions for Firebase のクライアント SDK を使用すると、クライアント側のFirebaseのWebアプリからCloud Functionsの関数を直接呼び出すことができます。この方法でアプリから関数を呼び出すには、Cloud Functions において HTTPS 呼び出し可能関数を記述してデプロイし、アプリから関数を呼び出すためのクライアント ロジックを追加します。要するにクライアント側から直接 Cloud Functions 関数を叩ける機能です。

ブラウザ上で画像ファイルを選択すると画像ファイルはファイルオブジェクトになり、それを再度ファイルにすることができず、”ダウンロード”をしないとファイル化できません。よってCloud Functionsをブラウザで動かし、Cloud fireStoreに一旦保存します。
